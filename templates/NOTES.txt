Thank you for installing {{ .Chart.Name }}.

ScyllaDB is now starting up — a fresh {{ .Values.scylladb.replicaCount }}-node
cluster can take a few minutes on first boot. You can watch it become ready:

    kubectl get pods -n {{ .Release.Namespace }} -w

When the scylladb pods are Ready, run the readiness check. IMPORTANT: pass
--timeout so `helm test` waits at least as long as the validator does
({{ .Values.validator.timeoutSeconds }}s) — without it, helm aborts after its
default 5 minutes:

    helm test {{ .Release.Name }} -n {{ .Release.Namespace }} --logs --timeout {{ add (div .Values.validator.timeoutSeconds 60) 5 }}m

This prints a PASS/FAIL summary. If ScyllaDB did not come up in time, it prints
the startup diagnostics (pod status, events, and the ScyllaDB boot log)
instead, so we can see what went wrong.

Please copy the full output of that command and send it to your Arnica contact.

If ScyllaDB keeps crashing or the check fails, collect a full diagnostics
bundle (Kubernetes objects, events, and ScyllaDB logs — no application data)
and send it back:

    curl -fsSL https://raw.githubusercontent.com/arnica-io/scylla-readiness-check/main/test/collect-diagnostics.sh | bash -s -- -n {{ .Release.Namespace }} -r {{ .Release.Name }}

When you are done, clean everything up with:

    helm uninstall {{ .Release.Name }} -n {{ .Release.Namespace }}
    kubectl delete namespace {{ .Release.Namespace }}
